# Prompt基础技巧

掌握基础的Prompt技巧是有效使用大语言模型的关键。本文将介绍最重要和最实用的Prompt设计方法。

## 1. 清晰的指令设计

### 使用具体的动词

选择明确、具体的动词来描述期望的行为：

```
❌ 模糊指令：
"处理这个文档"

✅ 清晰指令：
"总结这个文档的主要观点，提取3个关键结论"
```

### 指定输出格式

明确告诉模型你想要什么样的输出格式：

```
请分析以下数据并以表格形式输出结果：

| 指标 | 数值 | 变化趋势 | 分析 |
|------|------|----------|------|
| ... | ... | ... | ... |

数据：[你的数据]
```

### 设置约束条件

通过约束条件控制输出的范围和质量：

```
请写一篇产品评测，要求：
- 字数：500-800字
- 包含：优点、缺点、使用体验
- 语调：客观中性
- 避免：过度营销语言
- 结构：引言-详细评测-总结
```

## 2. 上下文提供技巧

### 背景信息设置

提供充分的背景信息帮助模型理解任务：

```
背景：我是一家初创科技公司的产品经理，正在开发一款面向大学生的学习管理应用。

任务：请帮我设计用户调研问卷，了解大学生的学习习惯和痛点。

要求：
- 问题数量：10-15个
- 包含选择题和开放题
- 涵盖学习方式、时间管理、工具使用等方面
```

### 角色扮演

为AI分配特定的专业角色：

```
你是一位拥有15年经验的资深财务分析师。请以专业的角度分析以下公司的财务状况：

[财务数据]

请提供：
1. 财务健康度评估
2. 主要风险点识别
3. 改进建议
```

### 目标受众明确

指定内容的目标受众：

```
请解释区块链技术，目标受众是：
- 年龄：25-40岁
- 背景：非技术专业人士
- 需求：了解基本概念和应用场景
- 时间：5分钟阅读时间

要求使用通俗易懂的语言，避免技术术语。
```

## 3. 示例引导（Few-shot Learning）

### 单一示例

提供一个清晰的示例来指导输出格式：

```
请将以下产品描述改写为营销文案：

示例：
产品描述："这是一款蓝牙耳机，具有降噪功能"
营销文案："沉浸式音质体验，智能降噪技术让您专注于纯净音乐世界"

现在请改写："这是一款智能手表，可以监测心率"
```

### 多个示例

使用多个示例展示模式和变化：

```
请根据以下示例，为新产品生成slogan：

示例1：
产品：运动鞋
Slogan："每一步都是突破"

示例2：
产品：咖啡
Slogan："唤醒每个清晨的可能"

示例3：
产品：护肤品
Slogan："时光温柔，肌肤如初"

现在请为以下产品生成slogan：
产品：智能家居系统
```

### 对比示例

通过正反示例明确期望：

```
请写一份专业的邮件回复，参考以下示例：

✅ 好的示例：
"感谢您的询问。关于您提到的项目时间安排，我们预计在下周三前完成初步方案。如有任何疑问，请随时联系我。"

❌ 避免的示例：
"收到，知道了，应该没问题。"

现在请回复：客户询问产品交付时间
```

## 4. 思维链技巧（Chain of Thought）

### 步骤分解

引导模型展示推理过程：

```
请解决以下问题，并详细说明思考过程：

问题：一家餐厅每天接待200位客人，平均每位客人消费80元。如果提高服务质量后，客人数量增加25%，平均消费增加15%，每天收入增加多少？

请按以下格式回答：
1. 理解问题：[分析题目要求]
2. 计算原收入：[详细计算]
3. 计算新收入：[详细计算]
4. 计算增加收入：[最终结果]
```

### 逐步推理

对于复杂问题，要求逐步分析：

```
请分析这个商业决策，并逐步说明推理过程：

情况：公司考虑是否进入新市场

请按以下步骤分析：
步骤1：市场机会评估
步骤2：竞争环境分析
步骤3：资源需求评估
步骤4：风险因素识别
步骤5：综合建议

每个步骤都要说明分析依据和逻辑。
```

### 自我验证

让模型检查自己的答案：

```
请计算以下数学问题，并在最后验证答案：

问题：[数学题目]

解答步骤：
1. [步骤1]
2. [步骤2]
3. [步骤3]

验证：
- 检查计算是否正确
- 确认答案是否合理
- 重新验算关键步骤
```

## 5. 结构化输出技巧

### 使用标题和分点

```
请分析这个市场趋势报告，按以下结构输出：

# 市场趋势分析报告

## 1. 核心发现
- 发现1：[具体内容]
- 发现2：[具体内容]
- 发现3：[具体内容]

## 2. 详细分析
### 2.1 增长趋势
[分析内容]

### 2.2 市场机会
[分析内容]

## 3. 建议行动
1. 短期建议：[具体建议]
2. 长期建议：[具体建议]
```

### 表格格式

```
请比较以下三个方案，用表格形式输出：

| 方案 | 成本 | 时间 | 风险 | 收益 | 推荐度 |
|------|------|------|------|------|--------|
| 方案A | | | | | |
| 方案B | | | | | |
| 方案C | | | | | |

评估标准：
- 成本：1-5分（1最低，5最高）
- 时间：预估完成时间
- 风险：低/中/高
- 收益：1-5分（1最低，5最高）
- 推荐度：1-5星
```

### JSON格式

```
请提取以下文本的关键信息，以JSON格式输出：

{
  "title": "文章标题",
  "author": "作者姓名",
  "publish_date": "发布日期",
  "main_points": [
    "要点1",
    "要点2",
    "要点3"
  ],
  "conclusion": "主要结论"
}

文本内容：[你的文本]
```

## 6. 条件和约束技巧

### 条件分支

```
请根据不同情况给出建议：

如果预算充足（>10万）：
- 建议方案A：[详细说明]
- 预期效果：[效果描述]

如果预算中等（5-10万）：
- 建议方案B：[详细说明]
- 预期效果：[效果描述]

如果预算有限（<5万）：
- 建议方案C：[详细说明]
- 预期效果：[效果描述]

当前预算：[具体金额]
```

### 优先级设置

```
请制定项目计划，按优先级排序：

高优先级（必须完成）：
1. [任务1]
2. [任务2]

中优先级（重要但可调整）：
1. [任务3]
2. [任务4]

低优先级（时间允许时完成）：
1. [任务5]
2. [任务6]

项目信息：[项目描述]
时间限制：[时间要求]
```

### 排除条件

```
请推荐适合的解决方案，但需要排除：
- 成本超过预算的方案
- 实施时间超过3个月的方案
- 需要额外人力资源的方案

要求：
- 提供3个可行方案
- 说明每个方案的优缺点
- 给出最终推荐

具体需求：[详细描述]
```

## 7. 迭代和优化技巧

### 渐进式细化

```
第一步：请先给出一个大致的方案框架
第二步：基于框架，详细展开每个部分
第三步：检查方案的可行性并提出改进建议

任务：[具体任务描述]
```

### 多轮对话优化

```
这是我们讨论的第[N]轮，基于之前的对话：

之前确定的要点：
- [要点1]
- [要点2]
- [要点3]

现在需要进一步细化：[具体需求]

请保持与之前讨论的一致性。
```

### 反馈整合

```
基于以下反馈，请修改之前的方案：

反馈意见：
1. [反馈1]
2. [反馈2]
3. [反馈3]

修改要求：
- 保留有效的部分
- 针对性改进问题点
- 确保整体协调性

原方案：[之前的方案]
```

## 8. 特殊场景技巧

### 创意激发

```
请用头脑风暴的方式生成创意：

主题：[创意主题]

要求：
- 生成至少10个不同的创意
- 每个创意用一句话描述
- 包含常规和非常规思路
- 不要自我审查，鼓励大胆想象

格式：
1. [创意1] - [简短说明]
2. [创意2] - [简短说明]
...
```

### 问题诊断

```
请帮我诊断以下问题：

问题现象：[具体描述]

诊断步骤：
1. 问题分析：可能的原因有哪些？
2. 优先级排序：哪些原因最可能？
3. 验证方法：如何确认真正原因？
4. 解决方案：针对性的解决办法
5. 预防措施：如何避免再次发生？
```

### 决策支持

```
请帮我做出决策：

决策问题：[具体问题]
可选方案：[方案列表]

分析框架：
1. 决策标准：[重要因素]
2. 方案评估：[逐一分析]
3. 风险评估：[潜在风险]
4. 推荐方案：[最佳选择]
5. 实施建议：[执行要点]
```

## 9. 质量控制技巧

### 输出验证

```
请完成以下任务，并在最后进行质量检查：

任务：[具体任务]

完成后请检查：
✓ 是否完全满足了所有要求？
✓ 信息是否准确可靠？
✓ 逻辑是否清晰连贯？
✓ 格式是否规范统一？
✓ 语言是否符合目标受众？

如发现问题，请主动修正。
```

### 多角度审视

```
请从以下角度审视这个方案：

技术角度：
- 技术可行性如何？
- 有哪些技术风险？

商业角度：
- 商业价值是什么？
- 投资回报如何？

用户角度：
- 用户体验如何？
- 是否解决了用户痛点？

运营角度：
- 实施难度如何？
- 需要什么资源？

方案内容：[具体方案]
```

### 持续改进

```
基于这次的输出结果，请分析：

1. 哪些部分效果最好？为什么？
2. 哪些部分还有改进空间？
3. 如果重新做这个任务，会如何调整？
4. 对类似任务有什么建议？

这种反思有助于提升后续任务的质量。
```

## 常见错误和避免方法

### 1. 指令过于模糊

❌ 错误示例：
```
"帮我写点东西"
```

✅ 正确示例：
```
"请写一篇800字的产品介绍文章，介绍我们的新款智能手表，
重点突出健康监测功能，目标读者是25-40岁的健康意识人群。"
```

### 2. 缺少上下文

❌ 错误示例：
```
"这个方案怎么样？"
```

✅ 正确示例：
```
"我们公司正在考虑以下营销方案来推广新产品。
作为市场专家，请评估这个方案的可行性：
[详细方案内容]"
```

### 3. 期望不现实

❌ 错误示例：
```
"给我一个保证成功的商业计划"
```

✅ 正确示例：
```
"请帮我制定一个餐厅商业计划的框架，
包括市场分析、财务预测、风险评估等关键部分。"
```

## 实践建议

### 1. 从简单开始
- 先用基础技巧
- 逐步增加复杂性
- 观察效果变化

### 2. 建立模板库
- 收集有效的提示模板
- 针对不同任务类型
- 便于快速复用

### 3. 持续测试优化
- A/B测试不同版本
- 记录效果差异
- 不断改进完善

### 4. 学习他人经验
- 参考优秀案例
- 加入社区讨论
- 分享自己的发现

<Tip>
掌握这些基础技巧需要大量实践。建议从日常工作中的具体任务开始，逐步应用和优化这些方法。
</Tip>

<Warning>
不要试图在一个提示中包含太多技巧，这可能导致指令过于复杂。选择最适合当前任务的2-3个技巧即可。
</Warning>
