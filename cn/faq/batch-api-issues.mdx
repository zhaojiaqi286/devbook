## 并发限制

### Batch API的并发限制是怎样的？

Batch API 的并发限制与现有的每个模型并发限制是分开的。BatchAPI 引入了两种新的限制：

**文件限制：**
- 单个 Batch 文件中包含最多 50,000 个请求且不超过 100M
- 向量模型（Embedding-2、Embedding-3）Batch文件请求数量限制为不超过 10000 次

**队列限制：**
每个模型的 Batch 有最大排队限制。当达到请求队列上限时，请等待当前任务完成后再提交新任务。

| 模型 | Batch 队列限制 |
|------|----------------|
| GLM-4-Flash | 1000万次 |
| GLM-4-Air | 1000万次 |
| GLM-3-Turbo | 200万次 |
| Embedding-2 | 200万次 |
| Embedding-3 | 200万次 |
| GLM-4-Plus | 200万次 |
| GLM-4-0520 | 50万次 |
| GLM-4 | 50万次 |
| Cogview-3 | 3万次 |

## 过期处理

### Batch 的过期如何处理？

如果Batch未能及时完成，该批次将被标记为过期状态；批次中未完成的请求将被取消。对于批次中已完成的请求，用户可以通过文件获取，并且需要支付这些请求消耗的费用。

**过期处理机制：**
- 未完成的请求自动取消
- 已完成的请求可正常获取结果
- 按实际消耗计费

## 文件管理

### Batch 文件有哪些存储限制？

**存储限制：**
- Batch 文件最多上传1000个文件
- 系统只保留您的文件30天
- 过期后文件将自动删除，无法恢复

**注意事项：**
- 请及时下载重要结果
- 建议本地备份重要文件
- 关注文件过期时间

### 如何删除Batch 文件？

**删除方式：**
1. 前往Batch数据页面进行删除
2. 通过调用接口删除

**删除注意事项：**
- 删除后无法恢复
- 确认不再需要后再删除
- 可批量删除多个文件
